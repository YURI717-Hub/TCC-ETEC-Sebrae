
Sobre o Git x GitHub:

Primeiro vou explicar em Analogia esses dois conceitos.

Cenário: Usuário de WhatsApp 

O Git = Histórico de Conversas:
"Ele salva tudo que foi enviado (mensagens, fotos, vídeos) com data e hora.
Você pode voltar numa conversa antiga, ver o que foi dito e até restaurar algo que apagou."

Porém, ele salva apenas localmente, nesse caso, no celular. 

Usar apenas o Git para salvar projetos é útil, o que ele pode fazer:
⦁	Histórico de versões
⦁	Comparação de mudanças
⦁	Possibilidade de voltar atrás
⦁	Trabalhar com branches (testes paralelos)
⦁	Tudo sem internet e sem conta online


O GitHub = Backup na Nuvem:
"O GitHub é como o backup do WhatsApp no Google Drive ou iCloud.
Ou seja, além de ter o histórico no seu celular (seu computador), você envia tudo pra nuvem (GitHub) pra guardar com segurança e poder acessar de outro lugar."

Lembrando, só é possível ver o histórico de alterações no GitHub por causa do git, pois além de mandar o projeto no GitHub, você também esta mandando o histórico completo do que fez usando o Git.


Agora usando o Git + GitHub é uma ótima combinação, por que:
⦁	Backup na nuvem
⦁	Acesso ao projeto de qualquer lugar
⦁	Compartilhamento com outras pessoas
⦁	Trabalhar em equipe, com controle de quem muda o quê
⦁	Integrações com ferramentas (CI/CD, Deploy, etc.)

Obs.: Existem outras plataformas que interagem com o git, como GitLab ou Bitbucket(ou Github)

Agora vem o Exemplo:

Situação: Você tem um grupo no WhatsApp com você mesmo onde salva ideias (seu projeto).
Agora você quer fazer o backup desse grupo na nuvem (GitHub).


Como mandar o projeto no GitHub:
1.	Crie um grupo no WhatsApp (GitHub → repositório).
2.	Ative o backup no celular (Git init).
3.	Escolha o que enviar no backup (Git add).
4.	Confirme o envio (Git commit).
5.	Mande tudo pro Google Drive/iCloud (Git push pro GitHub).

Como alterar só arquivos específicos de um projeto no GitHub:
Você quer reenviar só algumas fotos da galeria pro backup, não tudo de novo.

1.	Você muda 3 arquivos (como escolher 3 fotos novas).
2.	Usa git add só neles — "adiciona essas 3 fotos ao backup".
3.	git commit — "confirmar que são essas que quero salvar".
4.	git push — "envia pro backup".

Assim, só essas "fotos/arquivos" são atualizadas no GitHub.

Maneiras para garantir a segurança da sua conta GitHub:

1.	Nunca salve sua senha do GitHub em PCs públicos.
2.	Use autenticação de dois fatores — como o WhatsApp com código SMS.
3.	Sempre saia da sua conta do GitHub, igual ao "sair do WhatsApp Web".
4.	Prefira usar SSH com senha, como um PIN de segurança.
5.	Se usar GitHub em outro PC, gere um token temporário — como um código de login que expira.
6.	Evite clonar projetos privados fora do seu computador.

Agora aplicar na pratica, sem analogias, como enviar o Projeto no GitHub e fazer futuras alterações de arquivos específicos:
Explicarei 3 maneiras de fazer isso:

1.	Fazer pelo Terminal do Vscode:
Antes Crie um repositório no GitHub

⦁	Entre no vscode e abra o seu projeto, ou seja, seguir o caminho ate o diretório onde esta o projeto.
⦁	Va na aba Terminal e mude para Git bash
⦁        (echo "# nome do projeto" >> README.md) - opcional podera ser feito de pois
⦁	Digite git init - Inicia o Git no projeto.
⦁	(Digite git add READ.md - Prepara todos os arquivos pra salvar.) - se feito o comando com echo
⦁	Digite git add . - Prepara todos os arquivos pra salvar.
⦁	Digite git commit -m "primeiro commit(mudança)" - Salva uma versão com mensagem.
⦁	Digite git branch -M main - Renomeia a branch para main.
⦁	Digite git remote add origin https://github.com/SEU_USUARIO/meu-projeto.git - Conecta com a sua conta do GitHub.
⦁	Digite git push -u origin main - Envia o projeto pro GitHub

Obs.: Você pode usar o comando "git status" para saber se os arquivos foram salvos pelo git add .. 
obs.2: Verifique se sua branch esta main ou master. 

Como alterar arquivos de um projeto do GitHub:
Ex.: 
Digite "git add caminho/arquivo1 caminho/arquivo2" ou git add .
Digite "git commit -m "Atualizei os arquivos X e Y""
Digite "git push"

Obs.: Não será necessário apagar o arquivo antigo para colocar o novo, ele fara a substituição conforme o nome do arquivo e a extensão.
